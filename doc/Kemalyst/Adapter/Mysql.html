def migrate(table_name, fields) #

Migrate is an addative only approach. It adds new columns but never delete them to avoid data loss. If the column type or size changes, a new column will be created and the existing one will be renamed to old_{tablename} then the data will be copied to the new column. You may need to perform insert queries if the migration cannot determine how to convert the data for you.

def prune(table_name, fields) #

Prune will remove fields that are not defined in the model. This should be used after you have successfully migrated the colunns and data. WARNING: Be aware that if you have fields in your database that are not apart of the model, they will be dropped!
